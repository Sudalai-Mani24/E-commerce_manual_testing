# E-Commerce Manual Testing Project (nopCommerce)

## Project Overview
This project demonstrates end-to-end **manual testing** performed on the nopCommerce demo e-commerce web application.  
The goal of this project is to simulate a real-world QA process including test planning, test design, execution, and defect reporting.



## Objectives
- Understand real-world application workflows  
- Design test scenarios and test cases  
- Perform manual functional testing  
- Identify and document defects  
- Demonstrate complete QA lifecycle  


## Application Under Test
- **Application:** nopCommerce Demo Store  
- **Type:** E-Commerce Web Application  
- **Platform:** Web (Desktop)  


## Scope of Testing

### In-Scope Modules
- User Registration  
- User Login  
- Product Search  
- Product Details Page  
- Shopping Cart  
- Checkout  
- Order Confirmation  
- User Account  
- Wishlist  

### Out-of-Scope
- Admin panel testing  
- Performance testing  
- Security testing  
- Automation testing

## Defect Summary

A total of **6 defects** were identified:

| Severity | Count |
|--------|------|
| Critical | 1 |
| Major | 2 |
| Minor | 3 |

### Key Defect Areas:
- Cart pricing update issue  
- Login validation missing  
- Checkout payment validation issue  
- Registration password validation  
- Product search usability issue  
- Wishlist feedback missing  



## Test Environment
- **Browser:** Google Chrome  
- **Platform:** Desktop  
- **Network:** Internet  



## Limitations
- Testing performed on demo application  
- Limited backend control  
- Some defects are simulated based on expected behavior  



## Conclusion
The application is functionally stable for core user workflows.  
Most identified issues are related to **validation handling and usability improvements**.

This project demonstrates:
- Test planning  
- Test case design  
- Defect reporting  
- End-to-end QA workflow
